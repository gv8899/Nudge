## Design Context

### Users
知識型工作者（工程師、設計師、PM 等），在以下場景使用 Nudge：
- **早上/開工時**：安排今日任務、檢視待辦
- **工作中**：臨時接到任務時快速加入，不中斷心流
- **收工/回顧時**：寫日誌、回顧完成進度
核心需求是「輕量推進」——不需要複雜的專案管理，只要每天知道該做什麼、做了什麼。

### Brand Personality
**效率、專注、放心**

- 工具該等你，不是追你——不主動打擾，在你需要時隨時可用
- 操作直覺、回饋即時，不讓人猜
- 資料安全、狀態可靠，用起來沒有焦慮感

### Aesthetic Direction
- **Warm minimal**：暖色調舊紙質感，避免冷冰冰的科技感
- **Paper texture**：背景帶有細微紙張顆粒（桌面版），營造手寫筆記的溫度
- **參考**：Heptabase 的乾淨、專注、不花俏，但 Nudge 更聚焦在任務管理而非學習
- **色彩系統**：全部來自 `globals.css` 的 design tokens，暖棕色為主色調，狀態色使用低飽和的塵土色階
- **字型**：Geist Sans（UI）＋ serif accent（哲學/文學語氣）
- **Icon**：Lucide icon set（Web + App 統一）
- **Dark mode 為預設**，light mode 同等重視

### Design Principles
1. **效率優先**：每個操作都該在最少步驟內完成，不加多餘確認或動畫
2. **安靜但可靠**：介面不搶注意力，但狀態變化要有清楚回饋
3. **一致性**：Web 和 App 共用同一套 design tokens、icon set、元件結構
4. **系統化色彩**：所有顏色必須來自 design token，禁止硬編碼 hex 或隨意挑 Tailwind 預設色
5. **手感溫度**：在數位工具中保留一點類比感（紙質紋理、暖色調、手繪裝飾）
