## 專案概要
- 名稱：TaskDone（工作目錄名稱）
- 定位：輕量型專案管理工具
- 核心目的：不是追蹤大專案，而是管理專案底下需要日常 follow-up 推進的事情
- 參考對象：Heptabase 的 Journal 功能
- 開發順序：先網頁版，日後再開發 App

## 產品需求（已確認）
- 單人使用，不涉及多人協作，但需記錄「球在誰手上」
- 任務是主角，每天的頁面是「今天要推進哪些任務」的快照（類似 Sunsama 模式）
- 任務狀態：通用固定狀態，可來回切換（不需自訂流程）
- 等待他人時不需標注等誰，只需知道「不在我手上」
- 一眼看到任務目前在哪個階段（主要），也能回顧歷程（次要）
- 提醒機制：手動設定（替任務設「X 天後提醒我」）
- 未完成任務可透過日曆拖移到其他天
- 任務結構是扁平的，用 tag 分類，不需要上層「專案」概念
- 每天頁面除了任務清單，也需要**流水筆記區**記錄當天事項
- 現況痛點：用 Heptabase Journal 手動管理，手動移動任務、週回顧找未完成任務再搬，流程繁瑣

**Why:** 使用者需要一個專注於日常推進、追蹤的工具，現有 Heptabase Journal 需要太多手動操作
**How to apply:** 設計時保持輕量、快速，聚焦在每日記錄與推進追蹤；任務為核心實體，日視圖為快照；自動化取代手動搬移
