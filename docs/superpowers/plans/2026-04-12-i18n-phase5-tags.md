# i18n Phase 5 — TagManager / TagPicker / TagBadge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 web 的 `tag-manager.tsx` / `tag-picker.tsx` / `tag-badge.tsx` 和 mobile 的 `tag_manager.dart` / `tag_picker.dart` 字串搬到 i18n canonical（新增 top-level `tags.*` namespace，共 12 個 key）。顏色名稱不動，card detail 本體不動。

**Architecture:** 照 Phase 4 的節奏：先 seed canonical（zh-TW 手寫、en/ja 直接填）→ `npm run i18n:sync` → `flutter gen-l10n`。然後 web 3 檔案 / mobile 2 檔案獨立 task 遷移。最後一個 task 收尾驗證。

**Tech Stack:** next-intl v4、Flutter gen-l10n、`AppL10n`、既有 `useTranslations` pattern。

**Spec：** `docs/superpowers/specs/2026-04-12-i18n-phase5-tags-design.md`
